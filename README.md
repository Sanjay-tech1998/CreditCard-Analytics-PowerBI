# 📊 Credit Card Analytics Dashboard -- Power BI Project

## 🔷 Project Overview

This project consists of two interactive Power BI dashboards:

1.  Credit Card Customer Report
2.  Credit Card Transaction Report

The objective of this project is to analyze credit card customer
behavior, revenue patterns, transaction trends, and key performance
indicators to help financial institutions make data-driven decisions.

------------------------------------------------------------------------

# 📌 1️⃣ Credit Card Customer Report

## 🎯 Objective

To analyze customer demographics, revenue contribution, transaction
behavior, and segmentation patterns.

## 📂 Dataset Requirements

The dataset includes the following key fields:

-   Customer ID
-   Gender
-   Age Group
-   Income Group
-   Education Level
-   Marital Status
-   State Code
-   Card Category (Blue, Silver, Gold, Platinum)
-   Total Revenue
-   Total Transactions
-   Transaction Amount
-   Quarter
-   Use Chip (Swipe / Chip / Online)

## 🛠 Tools & Technologies Used

-   Power BI Desktop
-   Power Query (Data Cleaning & Transformation)
-   DAX (Calculated Measures & KPIs)
-   Data Modeling (Relationships)
-   GitHub (Version Control & Documentation)

## 📊 Key KPIs

-   Total Revenue
-   Total Transactions
-   Total Customers
-   Average Transaction per Customer
-   Revenue Growth %
-   Customer Count by Age Group
-   Revenue by Gender
-   Revenue by Card Category
-   Revenue by Income Group
-   Revenue by State

## 📈 Key Insights

-   Blue card category generates the highest revenue.
-   Male customers contribute slightly more revenue than female
    customers.
-   Customers aged 46--55 generate the highest revenue.
-   High-income customers contribute significantly more to total
    revenue.
-   Swipe transactions generate the highest revenue compared to Chip and
    Online.
-   Certain states contribute the most revenue.

## ⚠ Challenges Faced

-   Handling calculated columns vs measures correctly.
-   Creating proper DAX measures for growth %.
-   Formatting slicers and maintaining dashboard design consistency.
-   Managing relationships between transaction and customer tables.
-   Handling aggregation issues in combo and waterfall charts.

## 💡 Recommendations

-   Focus marketing campaigns on high-income and 46--55 age group
    customers.
-   Promote premium cards (Gold & Platinum) to increase revenue per
    customer.
-   Improve online transaction experience to increase digital adoption.
-   Target low-performing regions with promotional campaigns.

# 📌 2️⃣ Credit Card Transaction Report

## 🎯 Objective

To analyze transaction volume, revenue trends, expense types, and
customer behavior across quarters.

## 📊 Dashboard Features

-   Total Transaction Amount
-   Transaction Count
-   Customer Count
-   Total Revenue
-   Average Satisfaction Score
-   Revenue & Transaction Count by Quarter
-   Revenue by Expense Type
-   Transaction Count by Type (Online / Swipe / ATM)
-   Revenue by Customer Job
-   Revenue by Card Category
-   Customer Acquisition Cost by Card Category

## 📈 Key Insights

-   Q4 has the highest transaction count and revenue.
-   Bills and Entertainment are top revenue-generating expense
    categories.
-   Online and Swipe transactions dominate transaction count.
-   Businessman and White-collar customers generate the highest
    revenue.
-   Blue card category has the highest acquisition cost but also highest
    revenue.

## 📊 Business Impact

This dashboard helps:

-   Identify high-value customers
-   Track revenue growth patterns
-   Analyze transaction behavior
-   Improve marketing segmentation
-   Optimize operational performance

# 🚀 How to Use

1.  Download the .pbix file.
2.  Open in Power BI Desktop.
3.  Refresh data (if connected to source).
4.  Use slicers (Quarter, Card Category, Gender) to interact with
    dashboard.
